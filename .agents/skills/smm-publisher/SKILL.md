# Автопублікатор SMM дописів (SMM Publisher)

Цей скіл призначений для періодичного запуску у фоновому режимі скрипта автопублікації дописів, підготовлених у форматі Markdown.
Скрипт (`smm_publish.py`) перевіряє дописи, встановлює тимчасовий статус `processing` перед відправкою для запобігання дублюванню та надсилає їх на n8n вебхук. Після отримання відповіді скрипт оновлює статус допису на `published` або `partial` (у разі помилок на деяких платформах) та записує успішні та неуспішні мережі.

## Налаштування (Setup)

Перед першим використанням скіла у проекті агент має виконати такі кроки налаштування:

1. **Створення віртуального середовища (`venv`):**
   Створіть віртуальне середовище всередині папки скіла (для ізоляції залежностей):
   ```bash
   python3 -m venv .agents/skills/smm-publisher/venv
   ```

2. **Встановлення залежностей:**
   Встановіть необхідні пакети через створений `venv`:
   ```bash
   .agents/skills/smm-publisher/venv/bin/pip install -r .agents/skills/smm-publisher/scripts/requirements.txt
   ```

3. **Налаштування файлу `.env`:**
   Переконайтеся, що в корені вашого проекту (не в папці скіла) створено та налаштовано файл `.env` із такими змінними:
   ```env
   # Шлях до директорії, де знаходяться файли дописів Markdown
   POSTS_DIR=./
   # URL вебхука n8n для прийому постів
   N8N_WEBHOOK_URL=https://your-n8n-instance/webhook/smm-post-input
   # Максимальна кількість постів для публікації за один запуск
   BATCH_SIZE=5
   ```

## Інструкція з використання для Агента

Для налаштування автоматичної публікації агент має запланувати періодичне завдання (cron job) за допомогою інструменту `schedule`.

### Запуск планувальника (Приклад)

Агент повинен використовувати інструмент `schedule` для встановлення періодичного запуску скрипта за допомогою `python` з нашого `venv`.

**Приклад виклику інструменту `schedule`:**
```json
{
  "CronExpression": "*/15 * * * *",
  "Prompt": "Запустити скрипт публікації дописів за допомогою venv: .agents/skills/smm-publisher/venv/bin/python .agents/skills/smm-publisher/scripts/smm_publish.py. Після запуску перевірити оновлені статуси дописів у POSTS_DIR та повідомити користувача про успішно опубліковані мережі або помилки."
}
```

### Обробка результатів під час спрацювання планувальника

Коли заплановане завдання запускає агент:
1. Виконайте команду запуску скрипта:
   ```bash
   .agents/skills/smm-publisher/venv/bin/python .agents/skills/smm-publisher/scripts/smm_publish.py
   ```
2. Прочитайте вивід терміналу. Скрипт виведе лог обробки кожного файлу та відповіді від n8n.
3. Перевірте змінені файли Markdown у директорії дописів (`POSTS_DIR`).
4. Повідомте користувача про результати:
   - Які пости було опубліковано повністю (статус `published`).
   - Які пости опубліковано частково (статус `partial`) і які мережі потрапили до списку `networks_failed`.

## Обробка помилок (Error Handling)

1. **Зависання в статусі `processing`:**
   Скрипт захищений від зависання: у разі виникнення виключень (таймаут, помилка мережі) він автоматично повертає статус допису до початкового (`ready` або `partial`). 
   Якщо ви виявили файл із статусом `processing` без активного процесу відправки, це означає, що процес було примусово перервано (наприклад, зупинка контейнера). У такому разі агент має вручну повернути статус допису на `ready`.

2. **Помилки авторизації / Неправильний URL вебхука:**
   Якщо скрипт повідомляє про помилку 404 або 401/403, перевірте правильність `N8N_WEBHOOK_URL` у файлі `.env` у корені проекту.

3. **Відсутність відповіді (`⚪ Немає відповіді`):**
   Якщо n8n повертає відповідь `Немає відповіді` для мережі, яка була вказана у `networks` допису, скрипт автоматично позначить її у `networks_failed`. Агент має звернути на це увагу користувача, оскільки це свідчить про проблему з інтеграцією на стороні n8n.
